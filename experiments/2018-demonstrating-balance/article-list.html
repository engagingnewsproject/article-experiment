Go to Previous Article Link

Go to Previous Article link is dynamic and automatically appears if you go to a new page within the same session. A session is counted as one browser window/tab. As long as they're in the same browser window as when they started, their session will stay the same. When they click a link in the article, this new link will get added to their history we are storing with sessionStorage (basically Cookies, but for Javascript). The Go to Previous Article link will track every new page they are on, and remain showing until they have clicked it enough to go back to whatever page they started on.

Logs

For each new user, we generate a userID and store it in localStorage (like sessionStorage but persists across windows/tabs). This gets passed to the log for every interaction so we can see what a user does across their entire visit.

The breakdown of each log is like this:

[ url of the page they're on ]|[ id of the page ]|[ user ID ]|[ user IP ]|[ date logged ]|[ action category ]|[ action ]|[ description ]

For example, here's the log when a page is loaded:

https://demonstrating-balance.test/articles/stricter-gun-laws-prevent-shooting/balanced.php|r-db|db_5b180fd6d6814|127.0.0.1|06/06/2018 11:47:09 AM|Page View|Study:-Stricter-gun-laws-could-prevent-mass-shootings-and-save-lives|Loaded

Here are all the [ action category ]|[ action ]|[ description ]'s we are logging:

  • When a page gets loaded:
    Page View|[ url of the loaded page ]|Loaded
  • When a user clicks a link, we log the click before sending them to the new page:
    click|Link: [ link text ]|[ url of the destination the link goes to ]
  • When a Twitter for Facebook share button is clicked:
    click|Button: [ button type ]|[ button location ]
  • When a user fills out and successfully submits the comment form:
    Comment|Add Comment|[ name from comment form ]: [ comment text ]
